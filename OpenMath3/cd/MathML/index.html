The OpenMath/MathML3 Content Dictionaries

This directory contains the development versions of a set of Content Dictionaries, which declare and describe the set of symbols for MathML 3 (see the current wworking draft).

This CD set is under joint development of the W3C Math Working Group and the OpenMath Society

The CDs below are linked in their presentation. We also link

  • the XML source (*.ocd),
  • the OpenMath and MathML2 type information (*.sts and *.mts),
  • notation definitions (*.ntn).

General Discussions take place on the mailing list om3@openmath.org, you can self-subscribe at http://openmath.org/mailman/listinfo/om3. Please send your comments to the mailing list.

Specific discussions about issues will take place at https://trac.kwarc.info/OM3, again, you can self-subscribe.

  • arith1: This CD defines symbols for common arithmetic functions.
    Other Information: CD source (arith1.ocd) OM types (arith1.sts) MathML types (arith1.mts) Notations (arith1.ntn)
  • fns1: This CD is intended to be `compatible' with the corresponding elements in Content MathML. In this CD we give a set of functions concerning functions themselves. Functions can be constructed from expressions via a lambda expression. Also there are basic function functions like compose, etc.
    Other Information: CD source (fns1.ocd) OM types (fns1.sts) MathML types (fns1.mts) Notations (fns1.ntn)
  • linalg1: Operations on Matrices (independent of the matrix representation).
    Other Information: CD source (linalg1.ocd) OM types (linalg1.sts) MathML types (linalg1.mts) Notations (linalg1.ntn)
  • logic1: This CD holds the basic logic functions.
    Other Information: CD source (logic1.ocd) OM types (logic1.sts) MathML types (logic1.mts) Notations (logic1.ntn)
  • nums1: This CD is intended to be `compatible' with the MathML view of constructors for numbers (integers to an arbitrary base, rationals) and symbols for some common numerical constants. This CD holds a set of symbols for creating numbers, including some defined constants (i.e. nullary constructors).
    Other Information: CD source (nums1.ocd) OM types (nums1.sts) MathML types (nums1.mts) Notations (nums1.ntn)
  • relation1: This CD holds the common arithmetic relations. It is intended to be `compatible' with the appropriate MathML elements.
    Other Information: CD source (relation1.ocd) OM types (relation1.sts) MathML types (relation1.mts) Notations (relation1.ntn)
  • set1: This CD defines the set functions and constructors for basic set theory. It is intended to be `compatible' with the corresponding elements in MathML.
    Other Information: CD source (set1.ocd) OM types (set1.sts) MathML types (set1.mts) Notations (set1.ntn)
  • veccalc1: This CD contains symbols to represent functions which are concerned with vector calculus.
    Other Information: CD source (veccalc1.ocd) OM types (veccalc1.sts) MathML types (veccalc1.mts) Notations (veccalc1.ntn)
  • calculus1: This CD is intended to be compatible with the calculus operations in Content MathML. Integration is just for the univariate case and is either definite or indefinite.
    Other Information: CD source (calculus1.ocd) OM types (calculus1.sts) MathML types (calculus1.mts) Notations (calculus1.ntn)
  • integer1: This CD holds a collection of basic integer functions. This CD is intended to be `compatible' with the corresponding elements in Content MathML.
    Other Information: CD source (integer1.ocd) OM types (integer1.sts) MathML types (integer1.mts) Notations (integer1.ntn)
  • linalg2: This CD treats matrices and vectors in a row oriented fashion (using matrixrow's).
    Other Information: CD source (linalg2.ocd) OM types (linalg2.sts) MathML types (linalg2.mts) Notations (linalg2.ntn)
  • mathmlkeys: This content dictionary provides some keys for specifying the relations in semantics elements.
    Other Information: CD source (mathmlkeys.ocd) OM types (mathmlkeys.sts) MathML types (mathmlkeys.mts) Notations (mathmlkeys.ntn)
  • rounding1: A CD of basic rounding concepts
    Other Information: CD source (rounding1.ocd) OM types (rounding1.sts) MathML types (rounding1.mts) Notations (rounding1.ntn)
  • setname1: This CD defines common sets of mathematics
    Other Information: CD source (setname1.ocd) OM types (setname1.sts) MathML types (setname1.mts) Notations (setname1.ntn)
  • complex1: This CD is intended to be `compatible' with the MathML view of operations on and constructors for complex numbers.
    Other Information: CD source (complex1.ocd) OM types (complex1.sts) MathML types (complex1.mts) Notations (complex1.ntn)
  • interval1: This CD holds symbols which describe both discrete and continuous 1-dimensional intervals (with open/closed end points).
    Other Information: CD source (interval1.ocd) OM types (interval1.sts) MathML types (interval1.mts) Notations (interval1.ntn)
  • linalg3: This CD treats matrices and vectors in a column oriented fashion (using matrixcolumn's).
    Other Information: CD source (linalg3.ocd) OM types (linalg3.sts) MathML types (linalg3.mts) Notations (linalg3.ntn)
  • minmax1: This CD holds the definitions of min and max.
    Other Information: CD source (minmax1.ocd) OM types (minmax1.sts) MathML types (minmax1.mts) Notations (minmax1.ntn)
  • piece1: This CD is intended to be compatible with the corresponding elements in Content MathML 2. In this CD we give a set of operators for piece-wise defined expressions.
    Other Information: CD source (piece1.ocd) OM types (piece1.sts) MathML types (piece1.mts) Notations (piece1.ntn)
  • error1:
    Other Information: CD source (error1.ocd) OM types (error1.sts) MathML types (error1.mts) Notations (error1.ntn)
  • limit1: This CD holds the basic notion of the limits of unary functions whilst its variable tend (either from above, below or both sides) to a particular value.
    Other Information: CD source (limit1.ocd) OM types (limit1.sts) MathML types (limit1.mts) Notations (limit1.ntn)
  • list1: This CD is intended to be `compatible' with MathML list constructs.
    Other Information: CD source (list1.ocd) OM types (list1.sts) MathML types (list1.mts) Notations (list1.ntn)
  • multiset1: This CD defines the set functions and constructors for basic multiset theory. It is intended to be `compatible' with the corresponding elements in MathML i.e. set operations acting on sets of type=multiset.
    Other Information: CD source (multiset1.ocd) OM types (multiset1.sts) MathML types (multiset1.mts) Notations (multiset1.ntn)
  • quant1: This CD holds the definitions of the basic universal ("for all") quantifier and existential ("there exists") quantifier. It is intended to be `compatible' with the MathML elements representing these quantifiers.
    Other Information: CD source (quant1.ocd) OM types (quant1.sts) MathML types (quant1.mts) Notations (quant1.ntn)
  • s_dist1: This CD holds the definitions of the basic statistical functions used on random variables. It is intended to be `compatible' with the MathML elements representing statistical functions.
    Other Information: CD source (s_dist1.ocd) OM types (s_dist1.sts) MathML types (s_dist1.mts) Notations (s_dist1.ntn)
  • s_data1: This CD holds the definitions of the basic statistical functions used on sample data. It is intended to be `compatible' with the MathML elements representing statistical functions, though it does not cover the concept of random variable which is mentioned in MathML.
    Other Information: CD source (s_data1.ocd) OM types (s_data1.sts) MathML types (s_data1.mts) Notations (s_data1.ntn)
  • transc1: This CD holds the definitions of many transcendental functions. They are defined as in Abromowitz and Stegun (ninth printing on), with precise reductions to logs in the case of inverse functions. Note that, if signed zeros are supported, some strict inequalities have to become weak . It is intended to be `compatible' with the MathML elements denoting trancendental functions. Some additional functions are in the CD transc2.
    Other Information: CD source (transc1.ocd) OM types (transc1.sts) MathML types (transc1.mts) Notations (transc1.ntn)
